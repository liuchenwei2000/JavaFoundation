           应 用 程 序 打 包
           
当发布一个应用程序时，并不希望部署大量的类文件，类似于applet
应该把程序需要的相关类文件和其他资源文件打包成一个JAR文件
一旦将程序打包后，就可以通过一个简单命令加载它
如果正确地配置了操作系统，就可以通过双击JAR文件来加载

可以把应用程序、程序组件(beans)和代码库打包到JAR文件中
JAR文件是一个简单的ZIP文件，它包含类文件、程序需要的其他文件(如图标)
以及描述存档特性的清单文件(manifest)，该文件被命名为MANIFEST.MF
它在JAR文件的一个特殊的META-INF子目录下面

自运行Jar文件

需要在清单文件中添加一项来指定程序的主类(main class)该类一般是Java程序启动的入口类，如：

Main-Class：mypackage/MyClass

只有类名，不需要.class扩展名

密封包

如果在代码中使用包可见的类、方法或者域，那么可能希望对包密封
如果不密封，其他类就可以放置在这个包中，因此它们就可以访问包可见的特性
如果密封了mypackage包，那么就不能用下面的语句定义密封包外的类：

package mypackage

要实现这一点，需要把包中所有的类放到一个Jar文件中
在默认情况下，Jar文件中的包是没有密封的，可以在清单文件中加入下面一行：

Saled:true

来改变全局默认的设定

对于每个单独的包，可以指定是否密封该包。例如：

Name: mypackage
Saled: true
Name: mypackage2
Saled: false
